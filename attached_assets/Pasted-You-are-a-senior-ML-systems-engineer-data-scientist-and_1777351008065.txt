You are a senior ML systems engineer, data scientist, and technical documentation expert.

Your task is to generate a COMPLETE, PRODUCTION-GRADE documentation file named `analysis.md` for the `src/analysis` folder of a machine learning repository.

---

## 🎯 Objective

Create a deeply structured and technically precise document that enables any ML engineer, data scientist, or reviewer to:

* Fully understand the purpose of the analysis layer
* Interpret what each file computes and why
* Understand statistical, analytical, and diagnostic logic
* Trace how insights are derived from processed data
* Debug, validate, or extend analysis modules confidently

---

## 📁 Scope

Analyze ALL files inside:
src/analysis/

Include:

* Every Python file
* Subfolders (if any)
* Any shared utilities used within analysis

---

## 📚 Required Output Structure (STRICT)

### 1. Overview

* Purpose of `src/analysis`
* Role in ML lifecycle (EDA, diagnostics, feature understanding, validation)
* Difference between analysis vs features vs evaluation layers

---

### 2. Folder Architecture

* Tree structure of `src/analysis`
* One-line explanation per file/module

Example:
src/analysis/
├── eda.py              → Exploratory Data Analysis
├── stats.py            → Statistical summaries
├── drift.py            → Data drift detection
...

---

### 3. End-to-End Analysis Flow

* Step-by-step flow of how analysis is performed
* Input → intermediate computations → outputs (metrics, reports, plots)
* Function/module interaction graph (who calls what)
* Execution order if part of pipeline

---

### 4. File-by-File Deep Dive

For EACH file:

#### File: <filename.py>

**Purpose**

* Why this analysis exists (business + technical reasoning)

**Key Functions / Classes**
For each function:

* Function name
* Input parameters (types + meaning)
* Output (metrics, dicts, DataFrames, plots, etc.)
* Exact logic (step-by-step breakdown)
* Statistical or mathematical methods used

**Analytical Techniques Used**

* e.g., correlation analysis, hypothesis testing, distribution analysis, skewness/kurtosis, PCA, clustering, etc.

**Dependencies**

* Internal modules
* External libraries (numpy, pandas, scipy, statsmodels, etc.)

**Performance Notes**

* Vectorization
* Sampling strategies
* Memory-heavy operations

### 13. Example Usage

## ⚠️ Constraints

* DO NOT give generic explanations
* DO NOT skip any file
* DO NOT assume undocumented behavior
* Be mathematically and technically precise
* Avoid vague statements like “analyzes data”

---

## 💡 Style Guidelines

* Clean markdown with headings
* Use tables for metrics and schemas
* Use code blocks where needed
* Keep explanations concise but deep

---

## 🚀 Goal

After reading `analysis.md`, a developer should:

* Understand all analysis computations without reading code
* Know how to interpret outputs
* Be able to safely modify or extend analysis logic
* Trust the correctness and purpose of every metric
