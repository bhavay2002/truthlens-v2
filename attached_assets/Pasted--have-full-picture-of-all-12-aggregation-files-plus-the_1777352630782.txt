CRIT-10 — Smoothing operates on a mix of normalized and unnormalized weights
•	File / function: src/aggregation/weight_manager.py:184-199.
•	Issue: order of operations is:
i.	scale weights[k] *= scale (now unnormalized).
ii.	smooth weights[k] = (1-α)·self.weights[k] + α·weights[k] — self.weights[k] is the already-renormalized base (sums to 1 within group), weights[k] is the unnormalized scaled version (does not sum to 1).
iii.	renormalize per group.
•	Why fusion is incorrect: mixing normalized and unnormalized inputs in the convex combination → the smoothing factor α no longer means "10 % toward adaptive". It means "10 % toward an arbitrary-magnitude vector", and the renorm afterward erases the smoothing effect non-linearly.
•	Exact fix: renormalize before smoothing:
# 1. scale  →  weights = base * scale
# 2. renorm scaled
for grp in WEIGHT_GROUPS.values():
    total = sum(weights[k] for k in grp) + EPS
    for k in grp: weights[k] /= total
# 3. smooth (now both inputs sum to 1)
for k in weights:
    weights[k] = (1 - self.smoothing) * self.weights[k] + self.smoothing * weights[k]
# 4. final renorm (corrects rounding)

CRIT-12 — Multilabel entropy normalized as a Categorical → wrong magnitude
•	File / function: src/aggregation/aggregation_pipeline.py:317-351 (_compute_entropy) and src/aggregation/feature_mapper.py:166-182.
•	Issue: both functions do probs / sum(probs) then -Σ p·log p with max_ent = log(N). For multilabel sigmoid outputs (emotion, narrative, narrative_frame per config.yaml), each p_k is an independent Bernoulli — they should not be renormalized to sum to 1, and the maximum entropy is N · log(2), not log(N).
•	Why fusion is incorrect: an emotion vector [0.9, 0.9, 0.9, …] (all labels active, low uncertainty) is renormalized into a uniform Categorical → entropy = log(N) = max → reported as maximally uncertain when it is in fact maximally confident. •	Exact fix:
task_type = TASK_TYPES[task]   # from config
if task_type == "multilabel":
    p = np.clip(probs_arr, EPS, 1.0 - EPS)
    h = -np.sum(p*np.log(p) + (1-p)*np.log(1-p))
    ent_val = h / (p.size * np.log(2))
else:
    probs_arr = probs_arr / probs_arr.sum()
    h = -np.sum(probs_arr * np.log(probs_arr + EPS))
    ent_val = h / np.log(max(probs_arr.size, 2))
entropy[task] = float(np.clip(ent_val, 0.0, 1.0))
