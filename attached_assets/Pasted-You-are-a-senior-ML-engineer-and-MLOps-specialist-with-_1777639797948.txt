You are a senior ML engineer and MLOps specialist with deep expertise in training pipelines, optimization, and scalable model development.

Your task is to generate a COMPLETE, PRODUCTION-GRADE documentation file named `training.md` for the `src/training` folder of a machine learning repository.

---

## 🎯 Objective

Create a highly structured and technically rigorous document that enables any ML engineer to:

* Fully understand the training pipeline and workflow
* Reproduce the training process end-to-end
* Debug training issues effectively
* Tune hyperparameters and improve performance
* Safely modify or extend the training system

---

## 📁 Scope

Analyze ALL files inside:
src/training/

Include:

* Every Python file
* Subfolders (if any)
* Trainer classes, scripts, utilities, callbacks, schedulers

---

## 📚 Required Output Structure (STRICT)

### 1. Overview

* Purpose of `src/training`
* Role in ML pipeline (model learning phase)
* Relationship with:

  * `src/models`
  * `src/features`
  * `src/evaluation`
  * `src/config`

---

### 2. Folder Architecture

* Tree structure of `src/training`
* One-line explanation per file/module

├── scheduler.py         → Learning rate scheduling
...
