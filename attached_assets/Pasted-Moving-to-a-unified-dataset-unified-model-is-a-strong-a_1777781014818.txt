Moving to a **unified dataset + unified model** is a strong architectural decision—but it’s not just a merge operation. It fundamentally changes **data contracts, training dynamics, loss balancing, evaluation, and inference design**.

Based on your current TruthLens architecture , you are *already partially unified* (multi-task model + unified schema). The real shift now is toward **true end-to-end unification + tighter coupling between data, model, and aggregation**.

Let’s break this down precisely.

# 🧠 2. Required Changes — Dataset Level

## 2.1 Move from “Merged” → “Aligned”

Your current unified dataset is **column-aligned**, not **semantically aligned**.

### Problem:

* Bias dataset ≠ Emotion dataset ≠ Propaganda dataset
* Missing labels per row → sparse supervision

### Upgrade:

### ✅ A. Introduce **Task Presence Mask**

Add:

```python
task_mask = {
  "bias": 1/0,
  "emotion": 1/0,
  ...
}
```

### Why:

* Prevents noisy gradients when labels are missing
* Enables **partial supervision learning**

---

### ✅ B. Cross-Task Label Alignment

Create **derived signals**:

| Derived Feature      | From                   |
| -------------------- | ---------------------- |
| emotional_bias_score | bias + emotion         |
| propaganda_intensity | propaganda + narrative |
| ideological_emotion  | ideology + emotion     |

👉 This allows:

* **Cross-task supervision**
* Better shared representations

---

### ✅ C. Normalize Label Distributions Across Tasks

Current issue:

* Emotion → multi-label dense
* Propaganda → binary sparse
* Ideology → 3-class

👉 Leads to:

* Gradient imbalance

### Fix:

* Temperature scaling per task
* Label smoothing
* Class reweighting per dataset

## 3.1 Move from “Multi-Head” → “Interacting Heads”

Current:

```
Encoder → Independent Heads
```

### Upgrade:

```
Encoder → Shared Representation
        → Cross-Task Interaction Layer
```

### ✅ A. Add Cross-Stitch / Gated Sharing Layer

```python
H_shared = encoder(x)

```

👉 Heads influence each other


# ⚖️ 4. Required Changes — Loss Function

## 4.1 Replace Simple Sum Loss

Current:

```
L_total = sum(L_i)
```

### Problem:

* Dominant tasks overpower others

---

### ✅ A. Dynamic Task Weighting (CRITICAL)

Use:

#### Option 1: Uncertainty Weighting

```
L_total = Σ (1/σ_i^2) * L_i + log σ_i
```

#### Option 2: GradNorm

* Normalize gradient magnitudes across tasks

---

### ✅ B. Add Consistency Loss

Example:

```python
L_consistency = KL(bias_distribution || ideology_distribution)
```

👉 Forces logical coherence between tasks

---

### ✅ C. Add Representation Alignment Loss

```
L_align = cosine(H_bias, H_emotion)
```

👉 Encourages shared understanding
