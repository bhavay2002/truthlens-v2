## 🔧 Prompt: Generate `models.md` for `src/models`

You are a senior ML engineer, deep learning architect, and technical documentation expert with strong experience in building and deploying production-grade models.

Your task is to generate a COMPLETE, PRODUCTION-GRADE documentation file named `models.md` for the `src/models` folder of a machine learning repository.

---

## 🎯 Objective

Create a highly structured and technically rigorous document that enables any ML engineer or reviewer to:

* Fully understand model architecture and design choices
* Trace training, validation, and inference workflows
* Understand how inputs (features) are transformed into predictions
* Identify performance considerations and limitations
* Safely modify, retrain, or extend the model

---

## 📁 Scope

Analyze ALL files inside:
src/models/

Include:

* Every Python file
* Subfolders (if any)
* Model checkpoints, loaders, and utilities

---

## 📚 Required Output Structure (STRICT)

### 1. Overview

* Purpose of `src/models`
* Role in ML pipeline (features → predictions)
* Relationship with:

  * `src/features`
  * `src/training`
  * `src/inference`
  * `src/evaluation`

---

### 2. Folder Architecture

* Tree structure of `src/models`
* One-line explanation per file/module

Example:
src/models/
├── model.py              → Core model architecture
├── layers.py             → Custom layers
├── loss.py               → Loss functions
├── utils.py              → Helper utilities
...

---

### 3. Model Architecture

* Detailed description of model(s):

  * Type (e.g., Transformer, CNN, XGBoost, etc.)
  * Input/output structure
  * Layer-by-layer breakdown
  * Activation functions
  * Regularization techniques (dropout, batchnorm, etc.)

* Include:

  * Data flow through model
  * Diagram-style explanation (textual)

---

### 4. End-to-End Model Workflow

* Step-by-step:

  * Input features
  * Forward pass
  * Loss computation
  * Backpropagation (if applicable)
  * Output predictions

* Describe interaction with training/inference pipelines

---

### 5. File-by-File Deep Dive

For EACH file:

#### File: <filename.py>

**Purpose**

* Why this file exists

**Key Functions / Classes**
For each:

* Name
* Inputs (types + shapes)
* Outputs (types + shapes)
* Step-by-step logic
* Important implementation details

**Dependencies**

* Internal modules
* External libraries (PyTorch, TensorFlow, sklearn, etc.)

**Performance Notes**

* GPU usage
* Mixed precision (fp16/bf16)
* Memory considerations

---

### 6. Model Inputs & Outputs

* Input feature format:

  * Shape, type, preprocessing expectations
* Output format:

  * Probabilities, logits, labels, embeddings, etc.

---

### 7. Loss Functions & Optimization

* Loss function(s) used:

  * Formula + intuition
* Optimizer(s):

  * Adam, SGD, etc.
* Learning rate scheduling
* Regularization strategies

---

### 8. Training Integration

* How models are used in training:

  * Trainer interaction
  * Batch flow
  * Checkpointing

---

### 9. Inference Logic

* How predictions are generated
* Preprocessing before inference
* Postprocessing after inference
* Latency considerations

---

### 10. Model Checkpoints & Persistence

* How models are saved/loaded
* File formats (.pt, .pth, .bin, etc.)
* Versioning strategy

---

### 11. Config Integration

* Hyperparameters from config:

  * Learning rate, batch size, layers, etc.
* How configs control architecture or behavior

---

### 12. Optimization & Efficiency

* GPU/TPU usage
* Gradient checkpointing
* Batching strategies
* Inference optimization (quantization, pruning if any)

---

### 13. Extensibility Guide

* How to modify architecture
* How to add new model variants
* Where to plug custom layers or losses

---

### 14. Common Pitfalls / Risks

* Overfitting
* Vanishing/exploding gradients
* Data mismatch issues
* Incorrect input shapes
* Inference-training mismatch

---

### 15. Example Usage

* Minimal example:

  * Load model
  * Pass input
  * Get output

---

## ⚠️ Constraints

* DO NOT skip any file
* DO NOT give vague explanations
* DO NOT assume undocumented behavior
* Be precise and implementation-aware

---

## 💡 Style Guidelines

* Clean markdown formatting
* Use tables where helpful
* Use code blocks for clarity
* Keep explanations concise but deep
