# Training Guide

This document walks through **training the TruthLens AI model** from data preparation to a fully trained and ready-to-use model artifact.

---

## Prerequisites

Ensure the following are in place before training:

1. Python 3.12 environment with all dependencies installed (`pip install -r requirements.txt`)
2. Raw datasets placed in `data/raw/` under the appropriate task subdirectory
3. `config/config.yaml` reviewed and adjusted if needed
4. At least 8 GB of RAM (16 GB recommended for batch_size=8 on CPU)
5. A GPU with CUDA is strongly recommended for full training — on CPU, expect significantly longer run times

---

## Quick Start

To run the complete training pipeline with default settings:

```bash
python main.py
```

This will:
1. Load and clean raw datasets from `data/raw/`
2. Validate dataset quality
3. Merge datasets into a unified schema
4. Apply class balancing and data augmentation
5. Split into train / validation / test (70 / 15 / 15)
6. Train the MultiTask RoBERTa model
7. Evaluate on the validation set
8. Save the model to `models/truthlens_model/`

---

## Step-by-Step

### Step 1 — Prepare Raw Datasets

Place your datasets in the appropriate subdirectories:

```
data/raw/
├── bias/           # CSV files with columns: text, bias_label
├── emotion/        # CSV files with columns: text, emotion_joy, emotion_fear, ... (20 emotion columns)
├── ideology/       # CSV files with columns: text, ideology_label
├── narrative/      # CSV files with columns: text, hero_entities, villain_entities, victim_entities
└── propaganda/     # CSV files with columns: text, propaganda_label
```

**Minimum dataset requirements (from `data_config.yaml`):**
- Each row must have at least 30 words
- No more than 10% null values per column
- No more than 15% duplicate rows
- Each class label must appear in at least 10% of rows

**Supported datasets:**

| Task                   | Recommended datasets             |
|------------------------|----------------------------------|
| Bias detection         | BABE, BASIL, MBIC                |
| Emotion classification | GoEmotions, SemEval 2018         |
| Ideology detection     | AllSides                         |
| Narrative analysis     | FrameNet                         |
| Propaganda detection   | PTC Propaganda Corpus            |
| Fake news (optional)   | ISOT, LIAR, FakeNewsNet          |

---

### Step 2 — Review Configuration

Open `config/config.yaml` and verify:

```yaml
model:
  encoder:
    name: roberta-base    # Change to a larger model (roberta-large) for better performance
    max_length: 256       # Increase for long articles (at GPU memory cost)

training:
  epochs: 4               # Increase for better convergence
  batch_size: 8           # Reduce to 4 if you run out of GPU memory
  learning_rate: 2.0e-5   # Keep default unless hyperparameter tuning

  fp16: true              # Disable if not using CUDA

data:
  train_path: data/splits/train.csv     # These are auto-generated by the pipeline
  validation_path: data/splits/validation.csv
```

**Tip:** If you only have fake news data and no multi-task data, you can disable unused heads by removing them from `model.heads` in the config.

## Data Pipeline Internals

The training pipeline runs these steps in order:

```
1. src/data/load_data.py
   - Reads CSV files from data/raw/
   - Validates column presence

2. src/data/clean_data.py
   - Unicode normalization
   - URL and HTML removal
   - Contraction expansion
   - Lowercasing, whitespace normalization
   - Min word count filtering

3. src/data/validate_data.py
   - Null ratio check
   - Duplicate ratio check
   - Class balance check

4. src/data/merge_datasets.py
   - Merges all task datasets into unified schema
   - Aligns column names using data_config.yaml mappings
   → data/processed/unified_dataset.csv

5. Class Balancing
   - Oversampling (default) or undersampling or SMOTE

6. Data Augmentation
   - Synonym replacement
   - Random swap
   - Random deletion

7. src/data/data_split.py
   - Stratified split: 70 / 15 / 15
   → data/splits/train.csv
   → data/splits/validation.csv
   → data/splits/test.csv
```

## Hyperparameter Tuning

Enable automatic hyperparameter search using Optuna:

```yaml
# config/config.yaml
hyperparameter_tuning:
  enabled: true
  trials: 10
  direction: minimize
  metric: eval_loss
  search_space:
    learning_rate:
      min: 1e-6
      max: 5e-5
    batch_size:
      - 8
      - 16
    epochs:
      - 3
      - 4
```

Then run: `python main.py`

Optuna will run the specified number of trials and report the best hyperparameters found.

---

## Cross-Validation

Enable k-fold cross-validation:

```yaml
cross_validation:
  enabled: true
  splits: 5
  metric: eval_loss
```

---

## Resuming Training from a Checkpoint

If training is interrupted, resume from the latest checkpoint:

```yaml
training:
  resume_from_checkpoint: true
```

Checkpoints are saved every 500 steps (configurable via `logging.save_steps`) and up to 3 are retained.

## Training on GPU

**CUDA (NVIDIA GPU):**
```yaml
training:
  device: auto    # Automatically selects CUDA if available
  fp16: true      # Enables mixed-precision for faster training
```

Verify CUDA is available:
```python
import torch
print(torch.cuda.is_available())  # Should print True
```

**Apple Silicon (MPS):**
```yaml
training:
  device: auto
  fp16: false     # FP16 not supported on MPS
```

**CPU-only:**
```yaml
training:
  device: cpu
  fp16: false
  batch_size: 4   # Reduce batch size to avoid memory pressure
```

## Training from a Different Base Model

To use a larger or different pre-trained model, update the config:

```yaml
model:
  encoder:
    name: roberta-large           # Or: bert-base-uncased, distilroberta-base, etc.
    tokenizer_name: roberta-large
    max_length: 512               # Increase for longer context window
```

Note: `roberta-large` uses 24 transformer layers vs 12 for `roberta-base` — expect roughly 3× the training time and memory.

---

## Training Only Selected Task Heads

To train on a subset of tasks (e.g., only fake news and bias), remove unused head configurations from `config.yaml`:

```yaml
model:
  heads:
    bias_detection:
      num_labels: 3
      loss: cross_entropy
      label_column: bias_label
    # (remove ideology_detection, propaganda_detection, etc.)
```

Also ensure your dataset only contains the relevant task columns.

## Model Artifacts After Training

After successful training, the following files are created:

| Path                                 | Description                           |
|--------------------------------------|---------------------------------------|
| `models/truthlens_model/config.json` | Model architecture configuration      |
| `models/truthlens_model/tokenizer.json` | Tokenizer vocabulary and settings  |
| `models/truthlens_model/model.safetensors` | Trained model weights           |
| `models/tfidf_vectorizer.joblib`     | Fitted TF-IDF vectorizer              |
| `logs/training.log`                  | Full training log                     |
| `reports/evaluation_results.json`    | Final evaluation metrics              |
| `reports/confusion_matrix.png`       | Confusion matrix visualization        |

## Evaluation After Training

Evaluate the trained model on the test set:

```bash
python -c "
from src.evaluation.metrics import evaluate_model
results = evaluate_model('data/splits/test.csv')
print(results)
"
```

Metrics computed: accuracy, precision, recall, F1, ROC-AUC (for multi-label tasks), and calibration metrics.

Results are also written to `reports/evaluation_results.json` automatically after training.
